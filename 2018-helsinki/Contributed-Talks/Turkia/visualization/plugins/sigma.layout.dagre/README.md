    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#sigmalayoutsdagre" aria-hidden="true" class="anchor" id="user-content-sigmalayoutsdagre"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>sigma.layouts.dagre</h1>
<p>Plugin developed by <a href="https://github.com/sheymann">Sébastien Heymann</a> for <a href="https://github.com/Linkurious">Linkurious</a> and published under the <a href="/Linkurious/linkurious.js/blob/develop/plugins/sigma.layouts.dagre/LICENSE">MIT</a> license.</p>
<p>Contact: <a href="mailto:seb@linkurio.us">seb@linkurio.us</a></p>
<hr>
<p>This plugin implements a binding to the <a href="https://github.com/cpettitt/dagre">dagre layout</a>, which organises the graph using a DAG (directed acyclic graph) system, written by <a href="https://www.linkedin.com/in/chrismpettitt">Chris Pettitt</a>. It is especially suitable for DAGs and trees. For more information, please refer to its <a href="https://github.com/cpettitt/dagre/wiki">Dagre's documentation</a>.</p>
<p>Because Dagre is a separate library, the developer must manually include <a href="https://github.com/cpettitt/dagre/releases"><code>dagre.js</code></a> in their app in order to use the layout.</p>
<p><g-emoji alias="warning" fallback-src="https://assets-cdn.github.com/images/icons/emoji/unicode/26a0.png" ios-version="6.0">⚠️</g-emoji> The time complexity of the algorithm is quadratic and is thus suitable for small graphs with less than 300 edges only.</p>
<h2><a href="#recommended-reading" aria-hidden="true" class="anchor" id="user-content-recommended-reading"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Recommended reading</h2>
<p>source: <a href="https://github.com/cpettitt/dagre/wiki#recommended-reading">https://github.com/cpettitt/dagre/wiki#recommended-reading</a></p>
<p>This work was produced by taking advantage of many papers and books. If you're interested in how dagre works internally here are some of the most important papers to read.</p>
<p>The general skeleton for Dagre comes from <a href="http://www.graphviz.org/Documentation/TSE93.pdf"><em>Gansner, et al., "A Technique for Drawing Directed Graphs"</em></a>, which gives both an excellent high level overview of the phases involved in layered drawing as well as diving into the details and problems of each of the phases. Besides the basic skeleton, the author specifically used the technique described in the paper to produce an acyclic graph, and the author use the network simplex algorithm for ranking. If there is one paper to start with when learning about layered graph drawing, this is it!</p>
<p>For crossing minimization we used <a href="http://www.researchgate.net/profile/Petra_Mutzel/publication/30508315_2-Layer_Straightline_Crossing_Minimization_Performance_of_Exact_and_Heuristic_Algorithms/links/09e4150eabaf4cc7bd000000.pdf"><em>Jünger and Mutzel, "2-Layer Straightline Crossing Minimization"</em></a>, which provides a comparison of the performance of various heuristics and exact algorithms for crossing minimization.</p>
<p>For counting the number of edge crossings between two layers we use the <code>O(|E| log |V_small|)</code> algorithm described in <a href="http://jgaa.info/accepted/2004/BarthMutzelJuenger2004.8.2.pdf"><em>Barth, et al., "Simple and Efficient Bilayer Cross Counting"</em></a>.</p>
<p>For positioning (or coordinate assignment), we derived our algorithm from <a href="http://www.inf.uni-konstanz.de/algo/publications/bk-fshca-01.pdf"><em>Brandes and Köpf, "Fast and Simple Horizontal Coordinate Assignment"</em></a>. The author made some adjustments to get tighter graphs when node and edges sizes vary greatly.</p>
<p>The implementation for clustering derives extensively from Sander, <a href="http://scidok.sulb.uni-saarland.de/volltexte/2005/359/pdf/tr-A03-96.pdf"><em>"Layout of Compound Directed Graphs."</em></a> It is an excellent paper that details the impact of clustering on all phases of layout and also covers many of the associated problems. Crossing reduction with clustered graphs derives from two papers by Michael Forster, <em>"Applying Crossing Reduction Strategies to Layered Compound Graphs"</em> and <em>"A Fast and Simple Heuristic for Constrained Two-Level Crossing Reduction."</em></p>
<h2><a href="#methods" aria-hidden="true" class="anchor" id="user-content-methods"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Methods</h2>

<p>Changes the layout's configuration.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> listener <span class="pl-k">=</span> <span class="pl-smi">sigma</span>.<span class="pl-smi">layouts</span>.<span class="pl-smi">dagre</span>.<span class="pl-en">configure</span>(sigInst, config);</pre></div>

<h2><a href="#configuration" aria-hidden="true" class="anchor" id="user-content-configuration"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Configuration</h2>
<ul>
<li><strong>nodes</strong>: <em>array</em>: the subset of nodes to apply the layout.</li>
<li><strong>boundingBox</strong> <em>object|boolean</em>: constrain layout bounds. Value: {minX, maxX, minY, maxY} or true (all current positions of the given nodes).</li>
</ul>
<p><em>Algorithm configuration</em></p>
<ul>
<li><strong>directed</strong>: <em>boolean</em> <code>true</code>: if <code>true</code>, take edge direction into account.</li>
<li><strong>multigraph</strong>: <em>boolean</em> <code>true</code>: if <code>true</code>, allows multiple edges between the same pair of nodes.</li>
<li><strong>compound</strong>: <em>boolean</em> <code>true</code>: if <code>true</code>, allows ompound nodes, i.e. nodes which can be the parent of other nodes.</li>
<li><strong>rankDir</strong> <em>string</em> <code>TB</code>: direction for rank nodes. Can be <code>TB</code>, <code>BT</code>, <code>LR</code>, or <code>RL</code>, where T = top, B = bottom, L = left, and R = right.</li>
</ul>
<p><em>Easing configuration</em></p>
<ul>
<li><strong>easing</strong> <em>string</em>: if specified, ease the transition between nodes positions if background is <code>true</code>. The duration is specified by the Sigma settings <code>animationsTime</code>. See <a href="/Linkurious/linkurious.js/blob/develop/src/utils/sigma.utils.js#L723">sigma.utils.easing</a> for available values.</li>
<li><strong>duration</strong> <em>number</em>: duration of the transition for the easing method. Default value is Sigma setting <code>animationsTime</code>.</li>
</ul>
<h2><a href="#events" aria-hidden="true" class="anchor" id="user-content-events"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Events</h2>
<p>The plugin dispatches the following events:</p>
<ul>
<li><code>start</code>: on layout start.</li>
<li><code>interpolate</code>: at the beginning of the layout animation if an <em>easing</em> function is specified and the layout is ran on background.</li>
<li><code>stop</code>: on layout stop, will be dispatched after <code>interpolate</code>.</li>
</ul>
